# Security Program Roadmap (6-12 Months)

**Strategy:** Pragmatic implementation of core capabilities to move from Ad-hoc (Tier 1) to Risk-Informed (Tier 2).

## 1. GOVERN (GV)
*Establishing the organizational context, strategy, and policies.*
- [ ] **[GV.PO-01]** Formalize core Information Security Policy suite (ISO 27001 aligned).
- [ ] **[GV.RM-01]** Establish a simplified Risk Register foundation.

## 2. IDENTIFY (ID)
*Understanding the environment to manage risk.*
- [ ] **[ID.AM-01]** Establish Hardware/Software Asset Inventory baseline.
- [ ] **[ID.RA-01]** Conduct initial business impact analysis (BIA) for critical services.

## 3. PROTECT (PR)
*Safeguards to ensure delivery of critical infrastructure services.*
- [ ] **[PR.AA-01]** Enforce MFA on all remote access and administrative accounts.
- [ ] **[PR.DS-01]** Implement formal endpoint protection (EDR) across the fleet.

## 4. DETECT (DE)
*Identify the occurrence of a cybersecurity event.*
- [ ] **[DE.AE-01]** Enable centralized logging for critical authentication events.
- [ ] **[DE.CM-01]** Define baseline for anomalous activity alerting.

## 5. RESPOND (RS) & RECOVER (RC)
*Taking action regarding a detected cybersecurity incident and restoring capabilities.*
- [ ] **[RS.MA-01]** Draft and test a basic Incident Response Plan (IRP).
- [ ] **[RC.RP-01]** Validate backup restoration for critical business data.
